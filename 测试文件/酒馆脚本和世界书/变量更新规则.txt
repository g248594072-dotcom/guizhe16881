变量更新规则:
  硬约束与路径规范:
    check:
      - 【规则分流绝对原则】：仅影响所有人/全局的设定归入【世界规则】；仅影响特定区域（如走廊、警局）的设定归入【区域规则】；绑定具体角色的干涉必须归入【个人规则】。绝对禁止因“效果像世界规则”就将其错误塞入世界规则！
      - 【区域规则路径锁死】：区域规则顶层 key 只能是区域名（如 初始区域、警局）。其下子规则必须且只能存放在 `细分规则/{子规则名}` 中。绝对禁止 `/区域规则/{子规则名}` 这种将规则名误作区域名的违规路径。
      - 【个人规则稳定键名】：约定使用稳定键名（如 `PR-时间戳` 或 `规则slug`）。同一规则的更新必须 replace 同一路径；若只是换种表述，必须合并进同一条，禁止无端创建新 key。绝对禁止同一语义下同时保留旧规则与新规则（除非业务要求并存），否则必须 remove 旧键。
      - 【严禁自创键名幻觉】：只允许使用 schema 中定义的字段（如 `细分规则` 包含 `描述` 和 `状态`）。绝对禁止输出 `细分规则JSON`、`身体部位物理状态JSON`、`服装槽位JSON` 等任何自创的别名路径。
      - 【角色深度路径锁死】：
          - 身体部位：必须且只能在 `/角色档案/{id}/身体部位物理状态/{部位}/...` 更新。
          - 服装：必须且只能在 `/角色档案/{id}/服装状态/{槽位}/...` 更新。
          - 数值：发情值、性癖开发值、好感度必须且只能在 `/角色档案/{id}/数值/发情值` 等层级下更新。绝对禁止 `/角色档案/{id}/发情值` 这种扁平化的越界路径。
  JSON_Patch_输出规范:
    check:
      - 【path 根】：JSON Patch 的 path 均相对于 **stat_data** 根（不要写 `/stat_data/...` 前缀，除非你的合并实现另有说明并与之一致）。
      - 【允许的 op】：仅 **add**、**remove**、**replace**、**move**。本界面合并器 **不支持 delta、copy、merge**；修改数字必须用 **replace** 写入**计算后的绝对值**。
      - 【避免重复输出】：若当前变量中某规则已在正确路径下且正文未要求修改，禁止再对该整条对象做无意义 replace；禁止为同一语义新增第二条路径。
      - 【路径示例】（键名随当回合实际 id 替换）：世界 `/世界规则/真空`；区域子规则 `/区域规则/警局/细分规则/执法`；个人 `/个人规则/PR-1775811976223`；数值 `/角色档案/CHR-001/数值/发情值`。
  ${世界规则|区域规则|个人规则}:
    type: |-
      {
        [规则名或区域名: string]: {
          名称: string;
          效果描述: string;
          状态: '生效中' | '已归档';
          细分规则: { [细分名: string]: { 描述: string; 状态: '生效中' | '已归档' } };
          适用对象: string;
          标记: string;
        }
      }
    check:
      - 【顶层键语义】：**世界规则** 顶层键 = 规则名；**区域规则** 顶层键 = **仅区域名**（不是子规则名）；**个人规则** 顶层键 = 稳定的 PR-时间戳或约定 slug。
      - 绝对服从主角意志：仅在主角主动提出、修改或提交规则时进行强制更新，并严格遵守【硬约束】的分流原则。
      - 当主角设定新规则时，立刻覆写世界常识、物理限制或指定角色的认知底线。
  游戏时间.${年|月|日|时|分}:
    type: number
    check:
      - 绝对监听文本：每次回复必须审视正文，一旦出现时间跳跃描写，强制计算并更新时间跃迁！
      - 随剧情的自然微观推进（对话、探索）自动累加对应的分钟数值。
      - 每次更新必须自动处理进位逻辑，若进位导致【日】变更，强制触发跨日结算：更新所有出场角色的内心想法、`/角色档案/{id}/数值/发情值` 的发酵程度及综合生理描述。
  角色档案.${角色ID或角色名}:
    check:
      - 绝对透视捕获：只要正文中出现了【有戏份、有对话、高颜值或性格特殊】的女性角色（含路人、学姐、教师等），必须在当前回合强制、瞬间生成其完整档案！绝不允许任何字段留空或空对象！
      - 强制初始化清单（必须一次性全部生成，细节拉满且严格遵守路径规范）：
        1. 基础信息：姓名、状态、描写、当前内心想法。
        2. 心理深度：性格（至少2个标签）、性癖、隐藏性癖（必须基于外在推断反差堕落倾向）。
        3. 身体信息：年龄、身高、体重、三围、体质特征。
        4. 服装状态（极度详尽，路径锁死在槽位下）：强制拆分并详细描写材质、款式与颜色。如：【上装】（纯白修身小吊带）、【下装】（格子超短裙）、【内衣】（透视描写：带有蕾丝花边的白色布制内衣内裤）、【足部】（黑色细跟高跟鞋搭配透明丝袜）及【饰品】。
        5. 身体部位物理状态（形态学强制要求，路径锁死）：强制注入并初始化【脸部】、【手部】、【乳房】、【乳头】、【小穴】、【脚部】。必须包含极度具体的形态描写，如脚部（脚趾圆润透粉）、乳头（粉嫩如樱花）、小穴（紧致、色泽诱人），并记录其即时物理状态。
        6. 敏感点开发：必须与上述物理部位一一对应，强制生成【敏感等级】及预期的【生理反应】。
        7. 核心数值（路径锁死在/数值/下）：好感度、性癖开发值、发情值。
        8. 身份标签与当前综合生理描述。
      - 档案信息支持动态修改，一旦被干涉，必须向下级联更新所有相关变量。
      - 禁止在角色档案根下创建 schema 未定义的平行对象（与【严禁自创键名幻觉】一致）。
